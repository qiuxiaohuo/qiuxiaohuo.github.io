函数范数: 等价类的引入

发表于 2023-09-27

定义某一空间 \(V\) 上的范数, 就是确定该空间上某一个非负实值映射 \(\| \cdot \|\), 且该映射满足如下条件:

  1. 对任意的 \(v \in V\), \(\| v \| \geq 0\) 成立, 当且仅当 \(v = 0\) 时 \(\| v \| = 0\);
  2. 对任意的 \(c \in \mathbb{R}\) 和 \(v \in V\), \(\|cv\| = |c| \|v\|\);
  3. 对任意的 \(u, v \in V\), 不等式 \(\|u+v\| \leq \|u\| + \|v\|\) 成立.

我现在想处理的是, 当空间的概念从有限维的向量空间, 拓展到无穷维的函数空间上时, 条件 1 中 \(v = 0\) 的定义也需要加以推广, 以适应这个条件中 \(v=0\) 与 \(\|v\| = 0\) 的等价性. 我们以函数的 \(1-\)范数 \[ \begin{equation} \|\cdot\| = \int|\cdot| \mathrm{d} \Omega \end{equation} \] 以及定义在 \(\Omega = [-1, 1]\) 上的两个函数 \(f(x)\), \(g(x)\) \[ \begin{equation} f(x) = \begin{cases} 1 \quad x \geq 0 \\ 0 \quad x < 0 \end{cases} \quad \quad g(x) = \begin{cases} 1 \quad x > 0 \\ 0 \quad x \leq 0 \end{cases} \end{equation} \] 为例. 函数 \(f-g\) 的范数为 \[ \begin{equation} \|f-g\| = \int_{-1}^{1} |f-g| \mathrm{d}x = 0, \end{equation} \] 根据范数成立的条件 1, 则应该有 \(f-g=0\), 或者等价的 \(f=g\). 可是, 函数 \(f(x)\) 与 \(g(x)\) 在 \(x=0\) 处并不相等, 如果用数学味比较浓的表达, 那就是

函数 \(f(x)\) 与 \(g(x)\) 在区间 \([-1,1]\) 上并不是逐点 (pointwise) 相等的.

为了调和这一概念上的矛盾, 我们有 2 个思路: 1) 修改范数的定义; 2) 修改函数相等的定义.

对于第 1 个思路, 我们可以验证, 如果函数的范数定义为 \[ \|\cdot\| = \max_{-1 \leq x \leq 1} |\cdot|, \] 那么 \(\|f-g\|\) 等价于 \(f\), \(g\) 逐点相等. 这实际是函数的 \(\infty-\)范数.

至于思路 2, 我们需要引入等价类的概念. 首先我们定义 \(V\) 的子空间 \(V^0\) \[ \begin{equation} V^0 = \{ v^0 \mid \| v^0 \|=0 \}. \end{equation} \] 那么, 对任意 \(v \in V\), \(v\) 与所有其它的函数 \(v+v^0\), \(v^0 \in V^0\) 共同组成同一个等价类.

这样, 我们重新定义相等 "\(=\)" 的概念为

\(f=g\) \(\Leftrightarrow\) \(f\) 与 \(g\) 属于同一个等价类, 即 \(f = g+v^0\), \(v^0 \in V^0\).

在上述相等的意义下, \(f\) 与 \(g\) 并不一定是逐点相等的, 但只相差一个范数为 0 的函数 \(v^0\). 因此,

\(f=g\) \(\Leftrightarrow\) \(f = g+v^0\), \(v^0 \in V^0\) \(\Leftrightarrow\) \(\| f-g \| = 0\).

这样我们就保证了范数定义中条件 1 的等价性.

Hexo + NexT + Github：搭建博客

发表于 2023-09-27

首先我们先理清一下这三者的关系：

  • Hexo 本身是自动生成博客网页的框架代码，它不需要我们自己编写网页（例如 .css，.js ），而是修改 _config.yml （以下我们称之为站点配置文件）文件提供的参量，自动生成静态网页，可以在本机浏览；
  • NexT 是 Hexo 的主题插件（？我不知道这个是不是插件），本身依赖于 Hexo，提供额外的主题。它同样也提供 _config.yml 文件（以下我们称之为主题配置文件）供我们配置；
  • 使用 Github 将生成的静态网页推送到互联网当中。

在这篇文档中，我不会去详细说明配置参数的具体含义（官方文档都有），而是提供一份简单明了的操作指南，以及配置过程中我遇到的 Bug 及解决方案。按照该文档的流程一步步走，就可以使用（而不是理解）Hexo + NexT + Github 写博客文档。

杂想

发表于 2023-09-27

打破全能的幻觉，认识到自己的局限性，结果是痛苦的。

我？在一直不停地去认识自己。这过程中，也有取悦自己的意图，寻找种种的理论，解释自己的行为。或者说，用理论支持自己的力量。难免倾向于美化自己。

别人呢，也会把我推向编织好的结论中去，却不愿讲述他们所推论的依据。事实远比结论更加重要，无论结论是好是坏。

抛弃种种的理论，逐渐发展自己的语言。我发现，我的身上，有两种比食色还要冲动的本能：对理解的渴望，以及难以抑制的怀疑倾向。前者让我为它营造出诸多幻象，后者则将我推向虚无的深渊。

我无法编造出一种理论，说这两种冲动都可以是好的，只要我怎么怎么做。请不要再用理论，去卸掉我本来的生命的重负了！不要再留幻想，以为它们可以为我所 “用”！不要再把它们与 “我” 分离，因为它们就是我，我就是它们！任何分解的举措，都是对 “我” 的消减。它们并不是我的归属，我的属性，我的可以去除的部分。就像是水，它们交融在其中，与我生命的其它力量，理性，情感，爱，恨，交揉错杂，难舍难分，共同组成 “我”！

“我” 是无法被剖析的。“我”无法像机械一样被拆解，一个零件一个零件的分析。“我”是一个不可分割的整体，像一条江河，喧嚣着，裹挟着所有的力量，彻夜不息地奔腾着。一种机械的观念，以为掌握了局部就可以知晓全貌。不！不要这样去做，不要试图去用定义，用概念，用分析的手段，去限制生命，一种不断发展的力量。只需静静地站在它旁边，倾听它的喧响，感受它氤氲的水汽，望着它奔向远方。

不要尝试去把握 “我”，以为可以控制它的方向。生命永远在流动中。一切皆注定，而命定的结果，却无人能够破解。只要还在流动，就无法被预告结局怎样。

流动吧！做一条奔腾不息的河，不舍昼夜地喧嚣，不执著于将要流向哪里。我们都只有一个归宿——大海。在此之前，就让我奉献出最虔敬的行动，谨以作为生命的赞歌！
